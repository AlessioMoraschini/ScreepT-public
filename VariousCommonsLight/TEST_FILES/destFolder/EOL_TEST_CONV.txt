				<h2 id="subLauncherCompatibility" class="sectionSubTitle"> 1.1 - LAUNCHER/COMPATIBILITY</h2>[\n]
				<li><span class="highlightBoldedGrey">ScreepT is a ready to launch software</span> written in java, but, it has a built-in custom launcher meant to check the state of dependencies on your computer before to launch, and warn you if it found something missing (provinding different ways to install them). [\n]
				<p> [\n]
					<li><span class="highlightBoldedGrey">You can find the launcher</span> inside main folder once it's extracted from [\n]
					the given .zip archive, just use the correct one for your OS.[\n]
					Remember to download the right version for your OS! (The software is designed to run well under Windows, Linux, and MacOS, but it is possible to use it -with limited functions- under other OS with Java interpreter installed, AKA JVM/JRE/JDK ).<br><br>[\n]
					<span class="highlightBolded">Notice</span>: it's still possible to just launch the "ScreepT_Vxx.jar" that you find inside the "/application" folder, but remember: it's not designed to render and work the best this way... so you might have a non-optimal experience running the software this way.[\n]
					<br><br>[\n]
					[\n]
					<span class="highlightBolded">Notice</span>: full support is granted for java versions series 1.8.XX - 11.X.X : previous version will not work at all, while on newer versions some functions can be disabled or limited for compatibility reasons. [\n]
					You can either download Oracle's main version (<a class="dependencyLink" target="_blank" href="https://www.oracle.com/technetwork/java/javase/downloads/jdk11-downloads-5066655.html">JVM download page</a>) or openJdk version (<a class="dependencyLink" target="_blank" href="https://adoptopenjdk.net/releases.html">openJdk download page</a>), or download it from another source.[\n]
					<br>At the end of the process just refresh and/or restart ScreepT. During the launch process JVM will be searched automatically, but if it's not found you can specify manually the path of you JVM /bin folder ;).[\n]
					<br><br>[\n]
					[\n]
					After you run ScreepT_launcher executable you'll be lead to a basic panel where you can choose to start the software, select specific settings to run the software, or check the dependency.<br>[\n]
					If something is missing you'll be asked to update/download/install it (if you downloaded version without installers you'll need an internet connection for that). Once the required dependencies are installed, you can start the software, or if still having problems, try to restart the launcher.[\n]
					[\n]
					<p> You don't need to pass through the launcher panel each time... if you want, after the first launch is gone, you[\n]
						can disable launcher panel in the "general options" tab in ScreepT menu bar under the voice "preferences".[\n]
					</p>[\n]
				</p>[\n]

You can save, export, load, reset, reload your favourite configuration, or even use the default one. When selecting a specific configuration you specify different variables such as textEditor colors, text size, menu text size, graphic general style, type of encryption, default export/load folder, default image extension or preferences, and others like that.<br>[\n]

				<h2 id="funcScreeptSubMenu" class="sectionSubTitle"> 2.3 - SCREEPT SUBMENU</h2>[\n]
				<p><ul>[\n]
				[\n]
					<p>[\n]
						This is the core function of this software, and implements a multi crypter/decrypter of files and folders, with a key generator and custom key dictionary manager.[\n]
						<br>In this section you can find all the stuff you need to encrypt and decrypt the files you need to keep secret, using a custom algorithm and different kind of codings.[\n]
						<br><br>[\n]
						ScreepT uses AES-128 bit, combined with Steg-crypter, that can generate an image containing either cripter or raw bytes from the source stream selected, using a text-key for AES encoding, combined with a PNG image key.[\n]
						You can find information about how it works in the ScreepT section below :)[\n]
					</p>[\n]
					<span class="highlightBolded">IMPORTANT: </span>your source files will NOT be deleted or modified, only new ones will be created in the directory you choose to put them in.<br><br>[\n]
					[\n]
					<li><image class="iconImgTitleSect" src="../Icons/Menu_Icons/crypter.png"/> <span id="subSectTitleCrypter" class="subMenuthirdLevel">SCREEPT - CRYPT AND DECRIPT FILES</span></li><br>[\n]
						When opening this subPanel, you'll find:[\n]
						<ul class="subList2">[\n]
						[\n]
							<br><li><span class="highlightBolded">On the left side</span> you have informations about key Image-dictionary, a log for current operations, and a button to select a one-use key image importing it from your file system.</li>[\n]
							<br><li><span class="highlightBolded">On the right side</span> you have a drag 'n drop panel on which you can drag the files you want to code/decode, or select them using the given button "add source file or folder". After you load them you can add more content simply repeating this operation. The added files will appear in the "Selected files pool", in which you can see the files that will be crypted or decrypted(according to the button you click after).<br>[\n]
							<span class="highlightBoldedGrey">Notice</span> that until you crypt/decrypt them no file will be created.<br>[\n]
							By right clicking on this panel you can see a pop up with useful operations on the loaded files. Also this operations will not modify you source file, but only the link created to them.</li>[\n]
							<br><li><span class="highlightBolded">On the Top</span> you will find all informations about keys and files loaded, with a useful feedback message giving you information about missing keys or errors.<br>[\n]
							In this panel you can also provide the AES text key used to crypt/decrypt, and specify which type of coder you want to use. Please be sure to use the right mode for your needs, and remember that you need to use the same mode if you want to decrypt the crypted result!<br>[\n]
							<span class="highlightBoldedGrey">When you load a folder</span> the result will contain all his files, mantaining the directory structure.<br>[\n]
							<span class="highlightBoldedGrey">When using STEG/COMPLETE mode</span> each file coded will be a folder, for this reason when using this modes you can only decrypt folders. Be sure to import all the result folder when decrypting this kind of files!<br>[\n]
							<span class="highlightBoldedGrey">When decrypting files</span> you'll be asked to choose between putting them in a temporary on your folder (that will be automatically deleted as soon as you close the software), or use a custom directory you choose.</li>[\n]
							<br><li><span class="highlightBolded">In the central section</span> you'll find a text area that will print all informations about current process, and files/folders being crypted.</li>[\n]
						[\n]
						</ul><br>[\n]
						[\n]
						<h2 class="subList2">CRYPTER MODES: Description<h2>[\n]
						<p><ul>[\n]
							<li><span class="highlightBolded">AES_ONLY</span>[\n]
							This mode will use only the AES encryption algorithm, given a text key that you have to write in the password box on top panel. For top security, you should use a 16 characters-long password. <br>[\n]
							<span class="highlightBoldedGrey">IMPORTANT: </span> you will need both text password to decrypt the result crypted files, be sure to remember it!<br></li>[\n]
							[\n]
							<br><li><span class="highlightBolded">STEG_ONLY</span>[\n]
							This mode will use only the STEG encryption algorithm, given a PNG image key, that you have to specify in the left panel, either choosin' it from dictionary, or load it for one use only and then ScreepT will forget about it unless you choose again it. For top security, you should use a random pixels image that you can create with the given "Image Key Generator" that you find in ScreepT subMenu.[\n]
							<br>When using this mode, your files will be encoded one by one using the value of each pixel from key image. the result will be contained in as much images as needed to contain source file, for each file in the source pool, or contained in sosurce folders.<br>[\n]
							<span class="highlightBoldedGrey">IMPORTANT: </span> you will need the key image to decrypt the result crypted files, be sure to keep it in a safe place!<br>[\n]
							The number of images created depends on the size of source file and size of image: <br>[\n]
							 <ol><li>If you have to encode a large number of small files use a small image key, or the result will waste space.[\n]
							 <li>If you have to encode big files use a medium-big image, resulting in a faster encoding process.[\n]
							</ol>[\n]
							[\n]
							<br><li><span class="highlightBolded">COMPLETE</span>[\n]
							This mode will combine AES and STEG mode this way: first the source bytes from each file will be coded with AES encryption standard, and then they will be encrypted with STEG mode explained before.[\n]
							<br> When used with "Ultra secure" option, this grant a very secure encription by increasing exponentially the entrophy and the number of possible combination... making it near impossible to decrypt them without both text key and Image key.<br>[\n]
							<span class="highlightBoldedGrey">IMPORTANT: </span> you will need both text password and key image to decrypt the result crypted files, be sure to keep them in a safe place![\n]
							<br> The number of images created depends on the size of source file and size of image: <br>[\n]
							 <ol><li>If you have to encode a large number of small files use a small image key, or the result will waste space.[\n]
							 <li>If you have to encode big files use a medium-big image, resulting in a faster encoding process.[\n]
							</ol>[\n]
							[\n]
							<br><li><span class="highlightBolded">ULTRA SECURE OPTION</span>[\n]
							<p>In STEG/COMPLETE modes, when this checkbox is selected, files will have an extra phase in which source bytes of STEG phase will be reordered accorging to values contained in the given Key Image pixels: ScreepT will use theese values as "reorderKey index" to change the order oof source bytes before to crypt or decrypt them. <br>[\n]
							<span class="highlightBoldedGrey">IMPORTANT: </span>if you cript files with this mode, you have to select this option also when you'll have to decript the crypted result later. If not, decription won't succeed.[\n]
							</p>[\n]
						</ul></p>[\n]
